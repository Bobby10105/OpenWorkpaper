## 2026-06-18 - [Prisma Nested Creation Optimization]
**Learning:** [When creating multiple entities and their related children within an interactive Prisma transaction (`prisma.$transaction`), a sequential `for...of` loop causes N+1 query latency due to sequential `await` calls. While `createMany` is optimal for flat bulk inserts, complex nested creations across groups often require looping. Using `Promise.all` alongside `.map()` inside the transaction parallelizes these queries concurrently, resolving the N+1 latency.]
**Action:** [Next time I see sequential database insert logic in a `for...of` loop during a performance refactor, I will refactor it to a `Promise.all` using `map`, taking care to properly flatten and track return values if ordering needs to be preserved.]
